# Findings & Decisions

## Domain core architecture (2026-03-05)

- The app is domain-agnostic; domain behavior must come from pack config, not hardcoded language assumptions.
- Adopted projection-first layering over one canonical graph (nodes/edges remain single source of truth).
- Added config objects for `world`, `relation_kinds`, and per-node/per-edge `layer_membership`.
- Kept typed core fields and reserved JSON metadata/filter fields for extension.
- Dropped v1 import support to avoid dual-schema drift; loader accepts only pack `version: "2"`.

## Layer system split decision (2026-03-05)

- `layers` + `node_layers` are for node visibility projections.
- `connection_layers` + `edge_connection_layers` are for edge overlay projections.
- Node layer selection is single-active; connection layers are multi-active.
- Edge filtering now uses connection-layer memberships, not node-layer membership.

## Visual system decision (2026-03-05)

- Visuals are config-driven, not hardcoded per domain in frontend constants.
- Style precedence for edges: connection layer style > relation style > world default > hardcoded fallback.
- Style precedence for nodes: active node layer style > node-type style > world default > hardcoded fallback.
- Frontend still keeps interaction overrides (selected/pinned/neighbor/learned) above base styles for usability.

## Overlay root-cause decision (2026-03-05)

- Issue was stacking-context parent z-index, not child z-index values.
- Fix must target backdrop/root modal layer (`settings-backdrop`, `buffer-backdrop`).
- Standardized z-index hierarchy via CSS variables to prevent future drift.

## Black screen root causes (2026-02-27)

1. `postprocessing` npm package was missing — `EffectComposerPmndrs` requires it as a peer dep.
   Without it the effect composer replaces the render pipeline but outputs nothing → black canvas.
2. `Line2` from `three/examples/jsm` via `<primitive>` doesn't integrate cleanly with TresJS's
   render loop (custom shader attributes not set up automatically, silent shader failures).

## Line2 from @tresjs/cientos

- Points format: `[[x1,y1,z1], [x2,y2,z2]]` (array of tuples, not flat)
- Props: `points`, `color`, `lineWidth` (`:line-width` in template)
- No manual GPU lifecycle (geometry/material dispose) needed — cientos handles it

## TauRPC / bindings

- `src/bindings.ts` is auto-generated by Specta — never edit manually
- IPC proxy singleton lives in `src/composables/useTauRPC.ts`
