# Findings & Decisions

## Black screen root causes (2026-02-27)
1. `postprocessing` npm package was missing — `EffectComposerPmndrs` requires it as a peer dep.
   Without it the effect composer replaces the render pipeline but outputs nothing → black canvas.
2. `Line2` from `three/examples/jsm` via `<primitive>` doesn't integrate cleanly with TresJS's
   render loop (custom shader attributes not set up automatically, silent shader failures).

## Line2 from @tresjs/cientos
- Points format: `[[x1,y1,z1], [x2,y2,z2]]` (array of tuples, not flat)
- Props: `points`, `color`, `lineWidth` (`:line-width` in template)
- No manual GPU lifecycle (geometry/material dispose) needed — cientos handles it

## TauRPC / bindings
- `src/bindings.ts` is auto-generated by Specta — never edit manually
- IPC proxy singleton lives in `src/composables/useTauRPC.ts`
